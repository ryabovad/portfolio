# Портфолио Даши Рябовой

Mobile-first сайт, на этой итерации работает на 375px. Дальше — планшет 768 и десктоп.

## Структура папок

```
portfolio/
├── index.html              ← главная страница
├── content.json            ← контент главной (правишь его → меняется сайт)
├── README.md               ← вот этот файл
│
├── css/
│   └── style.css           ← все стили (общие для главной и кейсов)
│
├── js/
│   └── main.js             ← рендер из JSON и бургер-меню
│
├── fonts/                  ← ПОЛОЖИ СЮДА шрифт NauryzRedKeds (см. ниже)
│
├── images/
│   ├── avatar.jpg
│   ├── projects/
│   │   ├── fit-service.jpg
│   │   └── discomarket.jpg
│   ├── shots/
│   │   ├── crypto.jpg
│   │   └── analytics.jpg
│   └── cases/              ← картинки внутри кейсов (см. ниже)
│       ├── fit-service/
│       └── discomarket/
│
├── files/                  ← ПОЛОЖИ СЮДА resume.pdf (см. ниже)
│
└── cases/
    ├── fit-service.html    ← страница кейса FIT SERVICE
    └── discomarket.html    ← страница кейса Дискомаркет
```

## 1. Что нужно сделать перед запуском

### 🔠 Шрифт NauryzRedKeds (обязательно)

Положи в `fonts/`:
- `fonts/NauryzRedKeds.woff2` (рекомендую — грузится быстрее)
- `fonts/NauryzRedKeds.ttf` (фолбэк)

У тебя уже есть TTF — конвертацию TTF → WOFF2 можно сделать на [cloudconvert.com](https://cloudconvert.com/ttf-to-woff2) или [transfonter.org](https://transfonter.org/). Если положишь только TTF — тоже работает, просто грузится чуть медленнее. Без файла шрифта заголовки будут отображаться запасным (sans-serif).

### 📄 Резюме (опционально, до публикации)

Положи `resume.pdf` в папку `files/`. Если файла нет — кнопка «Резюме» в бургер-меню будет приглушённой и не кликабельной (так уже настроено, обновлять код не нужно). Когда положишь файл — открой `content.json` и поменяй:

```json
"resume": ""           ← заменить на:
"resume": "files/resume.pdf"
```

### 🖼️ Картинки внутри кейсов

В обоих кейсах сейчас стоят плейсхолдеры — серые блоки с подписью «такая-то картинка». На каждом плейсхолдере написан **точный путь**, куда нужно положить файл. Экспортируешь из Figma слой как PNG или JPG, переименовываешь, кладёшь в указанную папку — и плейсхолдер автоматически заменится картинкой.

**Полный список путей для FIT SERVICE** (`images/cases/fit-service/`):
- `hero-1.png`, `hero-2.png`, `hero-3.png` — три мокапа в верхнем блоке
- `process-1.png`, `process-2.png`, `process-3.png` — три мокапа перед «Процесс»

- `final-1.png` ... `final-6.png` — шесть финальных мокапов
- `figma.png` — декоративный скрин Figma в самом низу

**Полный список путей для Дискомаркета** (`images/cases/discomarket/`):
- `hero.png` — главный скрин маркетплейса
- `overview.png` — большой скрин под блоком «Аудитория»
- `feature-1-1.png`, `feature-1-2.png` — скрины фичи «минимальный заказ»

Сейчас на главной четыре UI-шота: Crypto App ✅, E-commerce App ⏳, Analytics Dashboard ✅, Task Manager ⏳. Двух не хватает — экспортируй из Figma и положи:

- `images/shots/ecommerce.jpg`
- `images/shots/tasks.jpg`

И в `content.json` поменяй пустые `"image": ""` на соответствующие пути:

```json

...
{ "image": "images/shots/tasks.jpg",     "label": "Task Manager",   "size": "short" }
```

## 2. Как запустить локально

Двойной клик по `index.html` **не сработает** — браузер заблокирует загрузку `content.json` и шрифтов из-за политики CORS на `file://`. Нужен локальный сервер.

### Вариант А — VS Code + Live Server (проще всего)

1. Открой папку `portfolio/` в VS Code
2. Установи расширение **Live Server** (от Ritwick Dey)
3. Правый клик по `index.html` → **Open with Live Server**
4. Откроется браузер на `http://127.0.0.1:5500` или похожем адресе

### Вариант Б — Python (в терминале)

```bash
cd portfolio
python -m http.server 8080
```

Открой `http://localhost:8080`. На Windows если `python` не работает, попробуй `py` или `python3`.

### Как смотреть в мобильном виде

В Chrome/Edge: **F12** → иконка телефона/планшета в левом верхнем углу панели (DevTools) → выбери iPhone или поставь ширину 375.

## 3. Как редактировать контент

### Главная (`content.json`)

Контент главной живёт в `content.json`. Открываешь его в любом редакторе, меняешь строчки, сохраняешь, обновляешь страницу — готово.

**Что можно менять там:**
- Имя, аватар
- Тексты Hero (заголовок, описание, теги B2C/B2B)
- Ссылки: Telegram, email, HH.ru, резюме (PDF)
- Проекты: добавляешь/удаляешь блоки в массиве `projects`. Каждый проект — это объект с полями (`title`, `company`, `period`, `image`, `about`, `role`, `metricsBlocks`, `url`)
- UI-шоты: массив `shots`. Размеры: `square` / `tall` / `medium` / `short`

**Формат метрик** — массив блоков, в каждом блоке 2 метрики (label + value):

```json
"metricsBlocks": [
  [
    { "label": "CR в запись",       "value": "+1.2%" },
    { "label": "Общий CR в запись", "value": "+0.2%" }
  ],
  [
    { "label": "Retention Rate", "value": "+ 3 %" },
    { "label": "CSAT",           "value": "+1.2"   }
  ]
]
```

Один блок = одна горизонтальная плитка с разделителем посередине. Если у проекта одна плитка метрик — оставляй один массив внутри. Если две (как у FIT SERVICE) — два массива друг под другом.

**Ссылка на кейс** — поле `url` в проекте. Если поле пустое (`""`), карточка отрисуется без ссылки. Если есть путь (`cases/fit-service.html`) — карточка кликабельна.

⚠️ JSON чувствителен к запятым и кавычкам. Если после правки сайт перестал загружаться — открой `content.json` в редакторе и проверь, что все скобки `{` `}` закрыты, нет «зависших» запятых перед `]` или `}`. Можно прогнать файл через [jsonlint.com](https://jsonlint.com/).

### Страницы кейсов (`cases/*.html`)

Кейсы лежат прямо в HTML — это удобнее, потому что они длинные и менять их будешь редко. Открываешь нужный файл, ищешь текст, меняешь, сохраняешь.

Чтобы поменять метрики в результатах FIT SERVICE — найди в HTML блок `<div class="case-results">` и правь значения внутри.

Чтобы поменять гипотезы в сетке — найди `<div class="case-hypotheses__grid">`. Зелёные гипотезы — класс `case-hypothesis--good`, отвергнутые — `case-hypothesis--bad`.
